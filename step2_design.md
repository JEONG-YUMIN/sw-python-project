# 설계 문서

## 1. 프로젝트 이름

시험 D-day와 공부 시간 관리 프로그램

## 2. 기능 목록

### MUST 기능

반드시 구현할 핵심 기능이다.

* 과목 등록 기능
* 시험 날짜 입력 기능
* 목표 공부 시간 입력 기능
* 현재 공부 시간 입력 기능
* 등록된 과목 목록 보기 기능
* 오늘 공부한 시간 추가 기능
* 시험까지 남은 D-day 계산 기능
* 목표 대비 공부 진행률 출력 기능
* 우선적으로 공부할 과목 추천 기능
* JSON 파일을 이용한 데이터 저장 및 불러오기 기능

### NICE 기능

시간이 되면 추가하고 싶은 기능이다.

* 공부 진행률을 막대 형태로 보여주는 기능
* 하루 평균 필요한 공부 시간 계산 기능
* 시험이 지난 과목 안내 기능
* 입력값 오류 처리 기능

### LATER 기능

이번 프로젝트에서는 구현하지 못했지만, 이후에 추가하고 싶은 기능이다.

* 등록한 과목 수정 기능
* 등록한 과목 삭제 기능
* 날짜별 공부 기록 저장 기능
* tkinter를 사용한 화면형 프로그램으로 개선
* 작은 D-day 창 또는 바탕화면 위젯 기능
* 프로그램 자동 실행 기능

## 3. 데이터 구조 설계

이 프로그램에서는 과목 하나의 정보를 딕셔너리로 저장한다.

```python
subject = {
    "name": "수리통계",
    "exam_date": "2026-06-20",
    "target": 15,
    "done": 3
}
```

각 항목의 의미는 다음과 같다.

* `name`: 과목명
* `exam_date`: 시험 날짜
* `target`: 목표 공부 시간
* `done`: 현재까지 공부한 시간

여러 과목은 리스트에 저장한다.

```python
subjects = [
    {
        "name": "수리통계",
        "exam_date": "2026-06-20",
        "target": 15,
        "done": 3
    },
    {
        "name": "파이썬",
        "exam_date": "2026-06-15",
        "target": 10,
        "done": 4
    }
]
```

전체 데이터는 다음과 같은 구조로 관리한다.

```python
data = {
    "subjects": subjects
}
```

프로그램이 종료되어도 데이터가 사라지지 않도록 `study_data.json` 파일에 저장한다.

## 4. 함수별 의사코드

### 1) 과목 등록 함수

```text
과목 등록 함수
1. 과목명을 입력받는다.
2. 시험 날짜를 입력받는다.
3. 목표 공부 시간을 입력받는다.
4. 현재까지 공부한 시간을 입력받는다.
5. 입력받은 정보를 딕셔너리로 만든다.
6. 과목 딕셔너리를 전체 과목 리스트에 추가한다.
7. 저장 함수를 실행한다.
```

### 2) 공부 시간 추가 함수

```text
공부 시간 추가 함수
1. 등록된 과목 목록을 출력한다.
2. 사용자가 공부 시간을 추가할 과목 번호를 선택한다.
3. 추가할 공부 시간을 입력받는다.
4. 선택한 과목의 현재 공부 시간에 입력한 시간을 더한다.
5. 변경된 정보를 저장한다.
```

### 3) 공부 현황 출력 함수

```text
공부 현황 출력 함수
1. 저장된 과목 목록을 불러온다.
2. 각 과목의 시험 날짜와 오늘 날짜를 비교하여 D-day를 계산한다.
3. 목표 공부 시간과 현재 공부 시간을 비교하여 부족 시간을 계산한다.
4. 목표 대비 진행률을 계산한다.
5. 과목별 D-day, 진행률, 부족 시간을 출력한다.
6. 부족 시간과 남은 날짜를 기준으로 우선 공부 과목을 추천한다.
```

### 4) 데이터 저장 함수

```text
데이터 저장 함수
1. 현재 과목 목록 데이터를 가져온다.
2. 데이터를 JSON 파일 형식으로 변환한다.
3. study_data.json 파일에 저장한다.
```

### 5) 데이터 불러오기 함수

```text
데이터 불러오기 함수
1. study_data.json 파일이 있는지 확인한다.
2. 파일이 있으면 저장된 과목 정보를 불러온다.
3. 파일이 없으면 빈 과목 목록으로 시작한다.
```

```text
[프로그램 시작]
        ↓

[메인 메뉴 출력]
        ↓

2. 오늘 공부 시간 추가
3. 공부 현황 보기

5. 종료
        ↓
[사용자가 메뉴 번호 선택]
        ↓
 ┌──────────────────────────────┐
 │ 1번 선택 → 과목 정보 입력       │
 │ 2번 선택 → 과목 선택 후 시간 추가 │
 │ 3번 선택 → D-day와 진행률 출력  │
 │ 4번 선택 → 등록된 과목 목록 출력 │
 │ 5번 선택 → 프로그램 종료        │
 └──────────────────────────────┘
        ↓
[기능 실행 후 다시 메인 메뉴로 돌아감]
```

## 6. 2단계 이후 구현 계획

2단계에서는 프로그램의 기능 목록, 데이터 구조, 함수 흐름, 화면 흐름을 설계하였다.

3단계에서는 먼저 과목 등록과 등록된 과목 목록 보기 기능을 구현하여 가장 작은 동작 버전을 만들 계획이다. 이후 공부 시간 추가, D-day 계산, 공부 현황 출력, JSON 저장 기능을 순서대로 추가할 예정이다.
